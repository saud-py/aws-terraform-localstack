# Lab 04: VPC, EC2, and Nginx Setup on LocalStack

This lab demonstrates how to provision a custom VPC, route tables, subnets, a security group, and an EC2 instance with user data to install Nginx.

## LocalStack EC2 Support Details

> [!WARNING]
> * **LocalStack Community (Free):** Only mocks the EC2/VPC APIs. Running this lab will succeed, and Terraform will show all resources as created, but no actual virtual machine or Docker container runs under the hood. Thus, you cannot SSH into the EC2 instance or browse the Nginx server.
> * **LocalStack Pro / Team:** Emulates EC2 instances using Docker containers. If you are running the Pro edition, LocalStack will start a Docker container representing your EC2 instance. The `user_data` script will execute to install Nginx, and you can SSH into it and browse the Nginx site.

---

## Architecture Components

* **VPC:** Custom 10.0.0.0/16 VPC.
* **Subnet:** Public subnet mapping public IPs on launch.
* **Internet Gateway & Route Tables:** Configured to allow ingress/egress.
* **Security Group:** Opens TCP ports 22 (SSH) and 80 (HTTP).
* **SSH Key Pair:** Automatically generated by Terraform (saved locally as `my-key.pem`).
* **EC2 Instance:** Launches an instance with a user data bootstrap script that installs and starts Nginx.

---

## Steps to Deploy

1. **Initialize Terraform:**
   ```bash
   terraform init
   ```

2. **Apply Configuration:**
   ```bash
   terraform apply -auto-approve
   ```
   *(This will create the resources and output `my-key.pem` private key in this folder.)*

3. **Verify Creation via AWS CLI:**
   Check that the instance is running and has an IP:
   ```bash
   aws --endpoint-url=http://localhost:4566 ec2 describe-instances
   ```

4. **Access SSH & HTTP (LocalStack Pro Only):**
   * **SSH:**
     ```bash
     ssh -i my-key.pem ubuntu@<instance-ip-or-dns>
     ```
   * **Browse Nginx:**
     Open your browser or use curl to access the public IP:
     ```bash
     curl http://<instance-ip>
     ```
